Welcome to Encryptify!

Symmetric Encryption

Uses a single key to securely scramble text through multiple cycles of obfuscation. If you use a key, you must provide the exact same key to decrypt it.

Asymmetric Encryption

Requires a numeric key of at least 3 digits to encrypt. It will generate two mathematical "roots" (e.g., 2.2, 5.2). To decrypt, you must provide these exact roots separated by a comma.

Algorithm Versions

Current Version (v2) [June 9, 2026 and later]: The default, modern secure algorithm.

Version 1 (v1) [Pre June 9, 2026]: Check the "Use Version 1 Algorithm" box only if you are trying to decrypt an older string that was encrypted with v1.

Limitations

Note: Encryptify is strictly designed for standard English characters and numbers. Encrypting special Unicode characters (such as emojis, Greek letters, or non-Latin alphabets) is not supported and will result in corrupted text upon decryption.
